- 個人利用のプロジェクトにつき、**後方互換性は考慮してはいけない。**
- シェルスクリプトはグローバル変数を利用しない。かならず関数の返値を利用すること。
- コメントは日本語、エラー文は英語
- コミットメッセージは規模に応じて「メインメッセージ」+「詳細3行まで」とする。
  - なるべく背景/意図を含めること
- コミット時は可能なら fast forward で行う
- 関数や定数には責務と内容を簡単にJSDocでコメント記述する(基本アノテーションなし)
- テストケースはプロダクションコードのそばに置き、命名は `*.spec.ts` とする
- コード変更後は基本的に `npm run check:all` をパスさせること
- knip 実行を指示された時だけ `npm run unused` で確認すること
- worktree-deck の設計判断・層責務・依存ルールは `docs/architecture.md` を参照し、実装とドキュメントの整合を維持すること
